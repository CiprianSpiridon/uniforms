# Mumzworld × Threads — School Uniform Landing Page

A self-contained landing page that **sells Threads school uniforms** on Mumzworld and
**cross-sells Mumzworld's own products**, styled to look native to mumzworld.com.

Flow: **pick school → pick year → pick child (gender) → tailored suggestions** (the exact
uniform + only the cross-sell that's appropriate for that age/gender).

## Run it

Just open `index.html` in a browser (no build step, no server — `data.js` is a plain
`<script>` include so it works from `file://`).

```
open index.html
```

## Files

| File | Purpose |
|------|---------|
| `index.html` | Markup: header, hero carousel, guided journey (4 steps), trust strip, footer, cart drawer |
| `styles.css` | Mumzworld design system (tokens mirror the real `mumzworld-nextjs` repo) |
| `app.js` | Vanilla JS: school autocomplete, year/gender gating, product cards, bundles, cart, carousels, EN/AR (RTL) toggle |
| `data.js` | Generated data bundle → `window.UNIFORM_DATA`. **Swap this for a live feed in prod.** |
| `data/*.json` | Source-of-truth data (see below) |
| `raw/` | Data pipeline: `catjson_raw.txt` (scraped), `build_data.js` (schools/years/tokens), `bundle.js` (builds `data.js`) |

## Data (`/data`)

- `schools.json` — **70 UAE schools** (from Threads `catjson`), with slug, logo, emirate, curriculum.
- `year-categories.json` — the **15 year levels** FS 1, FS 2, Year 1–13 (fixed taxonomy; per-school cat IDs vary).
- `filters.json` — gender + uniform-type facets.
- `sample-products.json` — real Threads uniform items (the demo catalogue).
- `cross-sell.json` — **APPROVED** Mumzworld cross-sell rails, each gated by `appliesTo.{years,genders}`.
- `mumzworld-categories.json` — Mumzworld category relevance map + Algolia notes (for live wiring).
- `design-tokens.json` / `landing-design.json` — extracted Mumzworld design system + landing components.

## Going live (Algolia)

The demo data are real SKUs. For production, replace `window.UNIFORM_DATA` (or just its
`crossSellRails[].items` / `uniformProducts`) with a live feed:

- **Index:** `magento2_aws_live_en_products` (brands: `magento2_aws_live_en_brands`)
- **Product URLs:** flat `/en/<slug>` (not `/p/`)
- **Cross-sell sourcing:** each rail lists `sourceCategories` (Mumzworld category URLs) — query those.
- **Creds:** Algolia appId/search key are proxied server-side on mumzworld.com and not client-exposed → get them from the backend team.
- **Uniform catalogue:** Threads is Magento; per-school year filter is `?cat=<id>` (IDs are per-school).

Regenerate `data.js` after editing any `data/*.json`:

```
node raw/bundle.js
```

> Note: `data/cross-sell.json` is hand-maintained/approved and is **not** regenerated by `raw/build_data.js`.
